# Bolt Share — Technical Implementation Plan

This document is the authoritative implementation plan for **Bolt Share** V1. It translates `.cursor/agent.md`, `.cursor/prompt.md`, and `.cursor/rules.md` into concrete build steps.

**Scope:** A small, production-ready WordPress shortcode plugin. No admin UI, database, AJAX, REST API, Gutenberg block, jQuery, or social SDK.

## 2. Plugin bootstrap and constants

### `bolt-share.php` responsibilities

1. WordPress plugin header (Name, Description, Version, Text Domain, Requires PHP, etc.).
2. `ABSPATH` guard — exit if accessed directly.
3. Define constants:
   - `BOLT_SHARE_VERSION` — semver string, also used for asset cache-busting.
   - `BOLT_SHARE_FILE` — `__FILE__`.
   - `BOLT_SHARE_PATH` — `plugin_dir_path( __FILE__ )`.
   - `BOLT_SHARE_URL` — `plugin_dir_url( __FILE__ )`.
4. Require `vendor/autoload.php` if present; otherwise register an admin notice and return early (no fatal).
5. Load text domain on `plugins_loaded` (priority before plugin boot, or within boot):
   ```php
   load_plugin_textdomain( 'bolt-share', false, dirname( plugin_basename( __FILE__ ) ) . '/languages' );
   ```
6. Instantiate `BoltShare\Plugin` on `plugins_loaded`.

**Nothing else** belongs in this file: no shortcode registration, no HTML, no SVG strings, no asset hooks.

## 4. Root Plugin service wiring

### `app/Plugin.php`

Single orchestrator class. No container, no interfaces for V1.

```php
namespace BoltShare;

use BoltShare\Frontend\Assets;
use BoltShare\Frontend\ShareShortcode;
use BoltShare\Support\SvgIcons;

final class Plugin {
    public function __construct() {
        $assets = new Assets();
        new ShareShortcode( $assets, new SvgIcons() );
    }
}
```

### Design decisions

- **Constructor injection** keeps dependencies explicit without a DI framework.
- `ShareShortcode` receives `Assets` so it can optionally trigger enqueue awareness later; in V1, `Assets` self-registers on `wp_enqueue_scripts` independently.
- `ShareShortcode` registers itself in its constructor via `add_shortcode()`.
- No singleton pattern — WordPress bootstraps once per request.

## 5. Frontend Assets class

### `app/Frontend/Assets.php`

| Responsibility | Implementation |
|---|---|
| Hook | `wp_enqueue_scripts` |
| Scope | Public frontend only (`! is_admin()` ) |
| CSS handle | `bolt-share-frontend` → `build/css/frontend.css` |
| JS handle | `bolt-share-frontend` (same handle for both, or separate handles with same prefix — use `bolt-share-frontend` for CSS and `bolt-share-frontend` for JS; WordPress allows duplicate handles across asset types) |
| Version | `BOLT_SHARE_VERSION` |
| JS placement | Footer (`true` in `wp_enqueue_script`) |
| Dependencies | None |
| Extension filter | `apply_filters( 'bolt_share_should_enqueue_assets', true )` |

### V1 enqueue strategy

Enqueue on **all public frontend requests**. Rationale:

- Shortcodes may appear in widgets, page builders, FSE templates, or dynamically injected content.
- Asset weight is tiny (~few KB combined).
- Avoids fragile "did the shortcode render?" detection.

### Instagram status strings

Pass translated fallback copy to JavaScript via `wp_localize_script()`:

```php
wp_localize_script(
    'bolt-share-frontend',
    'boltShareL10n',
    [
        'instagramCopied' => __( 'Link copied – open Instagram and paste it.', 'bolt-share' ),
    ]
);
```

This keeps JS free of hardcoded English while avoiding inline `<script>` blocks.

## 6. ShareShortcode class

### `app/Frontend/ShareShortcode.php`

#### Registration

```php
add_shortcode( 'bolt_share', [ $this, 'render' ] );
```

#### Render flow

1. Parse attributes via `shortcode_atts()` with filterable defaults.
2. Sanitize each attribute.
3. Resolve share URL and title (once per render).
4. Normalize network list (allowlist + dedupe).
5. Build per-network action data (Facebook URL, mailto URL, Instagram data attributes).
6. Generate unique panel ID: `bolt-share-panel-{increment}` using a private static counter on the class.
7. Load template via output buffering; return string.

#### Private methods (small, single-purpose)

| Method | Purpose |
|---|---|
| `get_defaults()` | Returns default attribute array; wrapped in `bolt_share_shortcode_defaults` filter |
| `sanitize_attributes( array $atts )` | Per-attribute sanitization |
| `resolve_share_url( string $url_attr )` | URL resolution with fallbacks |
| `resolve_share_title( string $title_attr )` | Title resolution with fallbacks |
| `normalize_networks( string $networks )` | Parse CSV, allowlist, dedupe; filter via `bolt_share_networks` |
| `build_facebook_url( string $url )` | Facebook sharer URL |
| `build_mailto_url( string $title, string $url )` | Encoded mailto |
| `get_panel_id()` | Unique ID generator |
| `render_template( array $data )` | `include` template with extracted vars |

#### Template data array (explicit contract)

```php
[
    'instance_id'     => 'bolt-share-panel-1',
    'label'           => 'Del',
    'dropdown_title'  => 'Del på...',
    'extra_class'     => 'my-custom-share',  // sanitized, may be empty
    'networks'        => [ 'facebook', 'instagram', 'email' ],
    'share_url'       => 'https://example.com/page/',
    'share_title'     => 'Page title',
    'facebook_url'    => 'https://www.facebook.com/sharer/sharer.php?u=...',
    'mailto_url'      => 'mailto:?subject=...&body=...',
    'icons'           => $this->svg_icons,  // SvgIcons instance
]
```

No magic globals. Template receives only what it needs.

## 7. SvgIcons allowlist class

### `app/Support/SvgIcons.php`

```php
public function get( string $name ): string
```

| Allowed `$name` | Usage |
|---|---|
| `share` | Trigger icon |
| `facebook` | Facebook row |
| `instagram` | Instagram row |
| `email` | E-mail row |

#### Rules

- Fixed SVG strings stored as private array or heredoc constants in the class.
- Unknown name returns empty string (never arbitrary input).
- Every SVG includes `aria-hidden="true"` and `focusable="false"`.
- SVG output is trusted plugin markup — echoed in template without `esc_html()` (would break SVG), but only from this allowlist.
- No filters accepting custom SVG in V1.

#### Icon sizing

SVGs use `width`/`height` attributes or CSS-controlled dimensions (~26×26px trigger icon). SCSS sets consistent row icon size in dropdown.

## 8. Template contract — `resources/templates/share.php`

### Input variables

Extracted from `$data` array passed by `ShareShortcode::render_template()`.

### Output structure

```html
<div class="bolt-share {extra_class}" data-bolt-share>
    <button
        type="button"
        class="bolt-share__trigger"
        aria-expanded="false"
        aria-controls="{instance_id}"
    >
        <span class="bolt-share__trigger-label">{label}</span>
        {share_icon}
    </button>

    <div
        class="bolt-share__panel"
        id="{instance_id}"
        data-bolt-share-panel
        hidden
    >
        <p class="bolt-share__title">{dropdown_title}</p>

        <ul class="bolt-share__list">
            <!-- facebook: <li><a> with href, target, rel -->
            <!-- instagram: <li><button data-bolt-share-instagram data-share-url data-share-title> -->
            <!-- email: <li><a href="mailto:..."> -->
        </ul>

        <p class="bolt-share__status" data-bolt-share-status aria-live="polite"></p>
    </div>
</div>
```

### Escaping rules per context

| Value | Function |
|---|---|
| Text content (label, title, network labels) | `esc_html()` |
| HTML attributes (class, id, aria-*, data-*) | `esc_attr()` |
| URLs in `href` | `esc_url()` |
| Pre-built mailto/Facebook URLs | `esc_url()` |
| SVG markup | Raw from `SvgIcons::get()` only |

### Network row labels (translated)

- Facebook → `__( 'Facebook', 'bolt-share' )`
- Instagram → `__( 'Instagram', 'bolt-share' )`
- E-mail → `__( 'E-mail', 'bolt-share' )`

### Progressive enhancement

- Facebook and e-mail are real `<a href="...">` links — functional without JS.
- Instagram is `<button type="button">` — requires JS for share/copy behavior.
- Dropdown toggle requires JS; closed panel remains in DOM with `hidden`.

## 10. URL and title fallback behavior

### Share URL resolution

```
1. If shortcode `url` attribute is non-empty after esc_url_raw():
     → use it (must be valid URL; empty result falls through)
2. Else if is_singular():
     → get_permalink( get_queried_object_id() )
3. Else:
     → home_url( '/' )
4. Apply filter: bolt_share_share_url
5. Final value passed through esc_url_raw() again before use
```

**Never** read `$_GET`, `$_POST`, or `$_SERVER['HTTP_REFERER']` for the share URL.

### Share title resolution

```
1. If shortcode `title` attribute is non-empty:
     → use sanitize_text_field() value
2. Else if is_singular():
     → get_the_title( get_queried_object_id() )
3. Else:
     → get_bloginfo( 'name' )
4. Apply filter: bolt_share_share_title
5. Strip tags; trim; fall back to translated "Share this page" if empty
```

Both URL and title are computed **once** per shortcode render and reused for Facebook, mailto, and Instagram data attributes.

## 13. Honest Instagram behavior

Instagram has **no reliable web share endpoint** for arbitrary URLs. V1 uses progressive enhancement only.

### Markup

```html
<button
    type="button"
    class="bolt-share__action"
    data-bolt-share-instagram
    data-share-url="{esc_attr share_url}"
    data-share-title="{esc_attr share_title}"
>
```

### JavaScript flow (`shareToInstagram()`)

```
User clicks Instagram button
  │
  ├─ navigator.share available?
  │    ├─ YES → await navigator.share({ title, url })
  │    │         ├─ Success → done (browser/OS picks target; may include Instagram)
  │    │         ├─ AbortError → silent return (user cancelled)
  │    │         └─ Other error → fall through to copy fallback
  │    └─ NO → copy fallback
  │
  └─ copyUrl()
       ├─ Try navigator.clipboard.writeText(url)
       └─ Else temporary <textarea> + document.execCommand('copy')
       └─ setStatus(translated "Link copied – open Instagram and paste it.")
```

### Honesty requirements

- Never open a fake Instagram URL pretending to share.
- Never show "Posted to Instagram" or "Opening Instagram".
- Status message only appears on copy fallback.
- `AbortError` from `navigator.share()` is **not** an error — no status, no console noise.
- Native share sheet target selection is **browser/OS controlled** — plugin cannot guarantee Instagram appears.

## 14. Accessible dropdown behavior

### Semantics

- Trigger: `<button type="button">` (not div, not `href="#"`).
- Panel: `<div hidden>` (not `aria-hidden` alone).
- No `role="menu"`, no roving tabindex, no focus trap.
- Actions are normal links/buttons in document tab order when panel is open.

### Interaction matrix

| Event | Behavior |
|---|---|
| Trigger click | Toggle panel; update `aria-expanded`; close other instances |
| Click outside | Close open panel; `aria-expanded="false"`; set `hidden` |
| Escape | Close panel; restore focus to trigger |
| Tab | Natural order; no trap |
| Open panel | Do **not** move focus into panel automatically |

### ARIA synchronization

- `aria-expanded`: `"true"` when open, `"false"` when closed.
- `aria-controls`: always points to panel `id`.
- Status region: `aria-live="polite"` on `[data-bolt-share-status]`.

### Focus

- Visible `:focus-visible` styles in SCSS — never removed without replacement.
- Icons decorative: `aria-hidden="true"`, `focusable="false"`.
- All options have visible text labels.

## 15. Multiple shortcode instances

### PHP

- Static instance counter on `ShareShortcode` generates unique IDs: `bolt-share-panel-1`, `bolt-share-panel-2`, etc.
- Each render is independent; no shared mutable state beyond ID counter.

### JavaScript

- Query all `[data-bolt-share]` roots on `DOMContentLoaded`.
- Track initialized roots in a `WeakSet` to prevent double-init.
- Each instance stores references to its trigger, panel, status element.
- **One** document-level click listener (delegated or single binding) for outside-click — attached once, not per instance.
- **One** document-level keydown listener for Escape — attached once.
- `closeOtherPanels(currentRoot)` runs before opening a panel.

### CSS

- Each `.bolt-share` is `position: relative` so its panel positions below its own trigger.

## 21. Definition of done

V1 is complete when **all** of the following are true:

1. Plugin activates cleanly with Composer autoload present.
2. Missing autoload shows clear admin notice; no frontend fatal.
3. `[bolt_share]` renders valid, escaped, accessible markup.
4. Visual design matches minimal "Del + share icon" reference.

6. Facebook and e-mail work without JavaScript.
7. Instagram uses honest progressive enhancement with copy fallback.
8. Multiple instances coexist on one page.
9. `npm run build` produces committed production assets.
10. PHP enqueues only from `build/`.
11. All automated checks in §20 pass.
12. No known security, accessibility, or release-blocking issues remain.

## 22. Explicit V1 exclusions

Do **not** implement unless scope is explicitly expanded:

| Excluded | Reason |
|---|---|
| Gutenberg block | Shortcode-only V1 |
| Admin settings page | No configuration needed |
| Options API / post meta | No persistence |
| Database tables | No data storage |
| AJAX / REST API | No server round-trips |
| jQuery | Vanilla JS only |
| Facebook/Instagram SDKs | No third-party scripts |
| Instagram OAuth / Graph API | No direct publishing |
| Analytics / tracking | Privacy and scope |
| Additional networks (Twitter/X, LinkedIn, etc.) | Allowlist is fixed at three |

## Overengineering review

This plan deliberately avoids:

- Dependency injection containers
- Interface/implementation pairs for each class
- Abstract base classes
- Separate "Network" strategy classes per platform (three networks fit cleanly in ShareShortcode + template conditionals)
- Custom post types, cron, transients
- Webpack/Gulp/@wordpress/scripts
- Feature flags or versioned migrations

**Appropriate complexity retained:**

- Three small PHP classes + one template (separation of concerns without ceremony)
- One JS entry with named helpers (readable, testable in browser)
- One SCSS file with BEM + custom properties (theme-friendly)
- Standard Composer PSR-4 (Bolt plugin convention)
- Document-level event listeners registered once (not per instance)

The architecture matches a ~500–800 LOC plugin, not a framework.

## Architecture summary

Bolt Share is a **thin-bootstrap, three-class PHP plugin** with a **single template**, **one JS bundle**, and **one CSS file**. The `[bolt_share]` shortcode parses attributes, resolves URL/title once, builds Facebook/mailto URLs in PHP, and renders accessible markup. JavaScript handles dropdown toggling and Instagram progressive enhancement only. Facebook and e-mail remain plain links that work without JS.

Data flows in one direction:

```text
Shortcode attrs → ShareShortcode (sanitize, resolve, build URLs)
                → share.php template (escape, markup)
                → Browser (JS: toggle, Instagram share/copy)
```
